## Introduction
Welcome to Module 4, Recruit! 
In the previous module, you encountered arrays when reading data from the Octoliner sensor using `octoliner.analog_read_all()`. You learned that an array (or list) can hold multiple numbers at once. 
Now, it is time to shift from *reading* lists generated by sensors to *creating* our own. 

In real-world space missions, rovers are rarely given commands one by one. Instead, Mission Control uploads a complete sequence of waypoints (a route) for the rover to execute autonomously. Today, we will use Python **Lists** to build a structured navigation path.

**Why use Lists?** Imagine navigating a complex crater with 50 different waypoints. Without lists, you would have to write movement and turning commands 50 times in a row! That is 100 lines of highly repetitive code. By storing those distances in a List, you only write the movement commands *once* inside a loop. The code remains exactly the same size whether you have 3 waypoints or 3000.

## Theory

### 1. Creating a List
To create a list in Python, enclose your values in square brackets `[]` and separate them with commas. 

```python
# A list containing three different turning angles
angles = [90, -45, 180]
```

### 2. A Reminder on Indexing
Just like with the Octoliner sensor data, every item inside a list has an index (its specific position), and in Python, counting always starts from exactly `0`.

You can extract a specific number by putting its index in brackets:

```python
first_angle = angles[0]  # This extracts '90'
```

### 3. List Length `(len())`
When a rover receives a route, it is good practice to confirm how many waypoints are in the package. You can find out how many items are inside a list using the built-in `len()` function (short for "length").

```python
total_commands = len(angles)
print("I have received", total_commands, "maneuvers.")
```

### 4. Iterating with a `for` Loop
In Module 2, you learned how to use a `for` loop to repeat an action. However, for loops have a superpower: they can iterate directly through a list!

Instead of using a numerical counter, Python will automatically take each item from your list, one by one, and place it into a temporary variable for you to use inside the loop.

```python
# The loop will run 3 times.
# "angle" becomes 90, then -45, then 180.
for angle in angles:
    robot.turn_right_angle(angle)
    time.sleep(0.5)
```
